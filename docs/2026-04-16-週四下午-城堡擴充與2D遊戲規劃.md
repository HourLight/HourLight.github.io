## 二、路線選擇（逸君需決策）

### 路線 A：擴充現有城堡系統（加料）

**優點：**
- 延續現有架構、使用者熟悉
- 工作量中等（3-5 小時完成擴充）
- 立即改善現有體驗

**風險：**
- 🔴 `hl-castle-material.js` 在「絕對不可碰清單」上
- 修改容易誤觸其他系統
- Sonnet 4 災難殷鑑不遠

**範圍：**
- 材料 55 → 100（+45 種）
- 家具 12 → 30（+18 個）
- Drop 來源新增：blog_read / aroma_garden / social_share / daily_checkin
- 每個房間對應 2-3 個專屬家具

### 路線 B：全新 2D 遊戲（保留現有城堡不動）⭐ 逸君傾向

**優點：**
- ✅ 現有城堡零風險
- ✅ 全新開發可以用現代 game framework
- ✅ 能做出更深的遊戲體驗
- ✅ 可以逐步迭代，不影響既有用戶

**風險：**
- 開發工作量較大（第一版可能需要 2-3 週陸續完成）
- 需要決定技術棧與美術風格
- 需要規劃如何跟現有 133 張牌卡 / 33 套命理整合

**建議技術棧：**
- **Framework**：[Phaser 3](https://phaser.io/)（最成熟的 2D Web Game Framework，社群大、文件完整）
- **語言**：原生 JavaScript（延續現有 codebase，不引入 React/Vue 避免複雜度）
- **美術**：手繪 cozy 風格（對應品牌溫暖調性）
- **存檔**：Firebase Firestore（與現有會員系統整合）
- **部署**：獨立 HTML 頁面（例如 `game-castle-v2.html`），不影響現有 castle-game.html

## 三、2D 遊戲概念框架（草案，週四下午需逸君確認）

### 3.1 遊戲主題（已鎖定）

**正式名稱**：**《馥靈城堡：覺察者之旅》**
**英文名**：**Hour Light Castle: The Awakener's Journey**
**短版稱呼**：馥靈城堡（四個字好記）

**核心故事**：
玩家是一位剛入門的「覺察學徒」，進入馥靈城堡尋找自己的人生鑰匙。
透過探索 12 個內在房間、收集 33 套命理線索、與寵物互動、完成每日覺察任務，
最終成為「馥靈傳承者」，解開屬於自己的人生座標。

**關鍵特徵**：
- 四個字主標 + 五個字副標（AAA 遊戲命名標準：薩爾達傳說：曠野之息 / 寶可夢：劍）
- 品牌錨定：「馥靈」兩字直接連結馥靈之鑰本尊
- 故事敘事：副標點明玩家身份（覺察者）與遊戲類型（旅程 = roguelike/rpg）

### 3.2 核心玩法（建議）

**日常循環（每天 5-10 分鐘）：**
1. 登入 → 領取今日鑰匙（隨機 3 把，對應 3 個房間）
2. 選一個房間進入 → 完成一個覺察小任務（抽牌 / 冥想 / 小測驗）
3. 任務完成 → 掉材料 + 獎勵靈感點
4. 寵物互動（餵食、擺設、擺姿勢）→ 情緒值提升
5. 材料夠了 → 合成家具 → 擺進房間
6. 分享任務結果 → 獎勵額外鑰匙

**進階玩法（長期）：**
- **季節事件**：24 節氣限定關卡（春分花祭、夏至火祭、秋分月祭、冬至雪祭）
- **命理挑戰**：解鎖 33 套命理後每套有一個「終極挑戰」
- **社群競賽**：週排行榜（覺察分、家具完成度、寵物親密度）
- **覺察師認證**：完成階段任務可獲得覺察師等級（對應現有 4 級認證）

### 3.3 與現有系統的整合

| 現有系統 | 整合方式 |
|---|---|
| 133 張牌卡 | 遊戲內「任務卡」系統，每張牌對應一個小任務 |
| 33 套命理 | 33 個命理「知識點」，解鎖後可用於遊戲內 buff |
| 101 項心理測驗 | 角色屬性面板（依 MBTI / 九型 / 愛之語等生成角色初始值） |
| 17 知識學苑 | 角色技能樹（學習芳療 / 經絡 / 水晶 / 塔羅解鎖對應技能） |
| 認知芳療 | 遊戲內「氣味記憶」機制（特定場景觸發芳療建議） |
| 城堡 12 房間 | 保留現有定義（H.O.U.R. + L.I.G.H.T.） |
| 會員系統 | 免費玩家基礎體驗，鑰友 / 大師解鎖高階內容 |
| PAYUNi | 遊戲內直接購買高階道具或 AI 解讀服務 |

### 3.4 美術風格（建議選項）

| 選項 | 風格 | 優點 | 缺點 |
|---|---|---|---|
| A. 像素藝術 | 8-bit / 16-bit | 復古感、檔案小 | 跟品牌調性不符 |
| B. 手繪 cozy | 水彩、溫暖色調 | ✅ 符合品牌 | 素材製作慢 |
| C. SVG 向量 | 幾何抽象 | 檔案小、RWD 友善 | 難做動畫 |
| D. AI 生成 | Midjourney / DALL-E | 快速產出 | 版權爭議、風格不統一 |

**建議**：**B. 手繪 cozy**（對應馥靈之鑰的淺色品牌 + 香氣溫暖感）
**折衷**：先用 AI 生成概念圖（D），再請逸君認識的插畫家（或用 Canva Pro）做最終版

### 3.5 技術架構草圖

```
hourlightkey.com/
├── game-castle-v2.html          ← 新遊戲主頁（獨立入口）
├── assets/game-v2/
│   ├── js/
│   │   ├── main.js              ← 遊戲主程式（Phaser 3 載入）
│   │   ├── scenes/              ← 場景（city/house/room×12/menu）
│   │   ├── systems/             ← 系統（inventory/quest/pet/dialogue）
│   │   └── data/                ← 遊戲資料（tasks/items/npcs）
│   ├── images/                  ← 美術素材
│   ├── audio/                   ← 音效 BGM
│   └── styles.css               ← UI CSS
└── api/game-v2-sync.js          ← Firestore 同步（如需新 API，但 Vercel 12 支上限）
```
