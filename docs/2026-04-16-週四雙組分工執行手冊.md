## 七、衝突處理 SOP

### Case 1：git push 被 reject

```bash
git push origin main
# 錯誤：Updates were rejected because the remote contains work that you do not have locally
```

**處理：**
```bash
git pull --rebase origin main
git push origin main
```

**⚠️ 如果 rebase 失敗（有衝突）：**
1. 立刻停手
2. `git rebase --abort`
3. Discord 喊：「B 組我 rebase 衝突，等妳先推」
4. 等對方推完再 `git pull` → 繼續

### Case 2：兩邊同時改同一個檔案

**絕對不要**。看第二節工作域隔離。如果真的不小心撞到：
1. 先停手的那個 revert 自己的改動
2. 兩邊在 Discord 討論誰先改
3. 改完的那邊推，另一邊 pull 後再改

### Case 3：共用檔案撞到（sitemap.xml 等）

用整點同步時宣告，寧可慢 5 分鐘等，不要兩邊同時改。

### Case 4：某個檔案改壞了

**不要硬修**，直接：
```bash
git revert <commit-sha>
git push origin main
```

在 Discord 喊：「我剛 revert 了 X commit，現有版本回到 Y」
